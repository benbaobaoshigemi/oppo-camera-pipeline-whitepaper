## 摘要

本文档对 OPPO Find X9 Ultra 三条主要拍照管线的底层实现进行完整的技术解剖：**普通拍照模式**、**大师模式（JPGmax）**以及**哈苏超清（50MP）**。

三条管线在分辨率路径、AI 降噪层和色彩渲染层三个维度上均存在本质差异，并非同一套管线的强度变体。大师模式 JPGmax 集成了 OPPO 自研 SimTool 引擎的全部六个色彩模块，以哈苏与 OPPO 联合调校、烧录于固件中的专属三维查找表作为最终色调映射核心，是本文档的重点研究对象。普通模式和哈苏超清的技术特征在相关章节中与大师模式形成对照，以揭示三者的架构差异与设计意图。

## 1. 背景与架构概述

### 1.1 三条管线的定义

Find X9 Ultra 的相机应用向用户提供三种主要的静态拍照路径，它们在传感器读出方式、AI 处理深度和色彩渲染架构三个维度上均存在本质差异。

**普通拍照模式**是设备默认的日常拍摄路径。以速度和便利性为优先目标，光线充足时输出约 25MP，弱光时退化为 12MP。色彩渲染由 ISP 硬件 Gamma 曲线和一层轻量的 LMT 查找表共同完成，不启用软件色调映射引擎的高级模块。

**大师模式 JPGmax** 是 Find X9 Ultra 面向追求极致成像质量的用户提供的旗舰拍照路径。"大师模式"指相机 UI 上的专业拍摄档位，"JPGmax"指该模式下 JPEG 输出采用最高画质编码，两者在技术层面绑定。大师模式的输出分辨率同样随光线条件自适应切换，光线充足时输出约 25MP，弱光时为 12MP。其核心优势不在分辨率，而在于启用了 SimTool 引擎的全部六个色彩处理模块，并使用由哈苏与 OPPO 联合调校的专属 3D LUT 作为最终色调映射层。

**哈苏超清（50MP）** 是 Find X9 Ultra 的高分辨率拍摄路径。该模式令传感器以全像素原生读出方式工作，不做任何像素合并，产出约 50MP 的图像。这条路径需要比其他模式大约八倍规模的 AI 降噪模型。关键在于：50MP 的色彩渲染层与当前所选拍摄档位完全共享——若在大师模式下触发 50MP，色彩由大师引擎处理；若在普通模式下触发，色彩由普通管线处理。50MP 是分辨率路径，不是独立的色彩系统。

### 1.2 OPPO SimTool 引擎

SimTool（Simulation Tool）是 OPPO 为 Find X 系列旗舰机型开发的软件色彩渲染引擎，运行于相机 HAL（硬件抽象层）之上，在 ISP 输出图像之后、JPEG 压缩编码之前执行。

SimTool 通过加载 .ini 格式的配置文件来决定启用哪些模块、使用哪些 LUT 文件。不同的拍照模式加载不同的 INI，从而呈现出完全不同的色彩特征。配置文件存放在固件 /odm/etc/camera/basictone/setting/ 目录中，LUT 数据文件分布在 /odm/etc/camera/basictone/ 及 /odm/etc/camera/filters_lut/masterlut/ 目录下。

### 1.3 LUT（查找表）的工作原理

LUT（Look-Up Table，查找表）是色彩处理系统的基础工具。其原理是：预先计算好每个输入颜色对应的输出颜色，并以大型表格的形式存储；运行时直接查表，而不需要实时执行复杂的数学运算，兼顾精度与性能。

本系统使用的 LUT 均为三维 LUT（3D LUT）：输入为红（R）、绿（G）、蓝（B）三个坐标轴，通过三线性插值在预存格点之间计算任意颜色的输出值。3D LUT 可以表达任意复杂的非线性颜色映射，包括改变色相、饱和度、亮度，乃至引入通道间的交叉耦合——即允许蓝色通道的输出依赖于红色通道的输入值，这是传统矩阵无法实现的能力。

LUT 文件以原始二进制格式（FP32，单精度浮点，每个数值占 4 字节）存储，内存排列使用 Fortran 顺序（R 轴变化最快，B 轴变化最慢），与业界通用 .cube 文件格式的 C 顺序方向相反。

## 3. 普通拍照模式管线

理解大师模式的最佳入口是先充分理解普通模式的结构，从而形成精准的对照。

### 3.1 分辨率路径：动态自适应的两档读出

普通模式同样存在两套分辨率路径，由 ISP 的场景亮度评估（Lux Index）自动选择：

**弱光路径（~12.5MP）**：在夜间、室内昏暗环境或高 ISO 场景下，传感器以 2×2 Binning 模式工作——将四个相邻同色物理像素在模拟域合并，等效像素面积扩大至约 2.0μm，大幅提升进光量和信噪比，代价是分辨率降至约 4096×3072（12.5MP）。对应降噪模型包括 photo_main_binning_evm.bix（2.6MB）等 inning 系列。

**日光路径（~25MP）**：在日光或充足人工光照下，传感器切换至 SQuadBayer 或混合读出（Mix）模式，输出约 5600×4200（~23.5MP，通称 25MP）。此模式保留了更多细节，同时因像素面积适中，信噪比仍处于可接受范围。对应降噪模型为 photo_main_mix_ev0_25M.bin（2.9MB）等 mix/squadbayer 系列。

普通模式的最高分辨率上限为 25MP，无法触发 50MP 全像素读出——后者专属于用户主动切换的哈苏超清模式。

### 3.2 AI 降噪模型

普通模式的降噪模型规模为 2.6–3.2MB，针对各传感器（主摄、超广角、3.5× 长焦、6× 超长焦）和各曝光条件（ev0、evm、evmm，分别对应标准、欠曝、深欠曝）分别训练了对应的专用模型，命名规则清晰反映了适用场景（如 photo_tele1_binning_evm_3x.bin 表示 3× 长焦镜头、低曝光、Binning 模式的降噪模型）。

### 3.3 SimTool 阶段：仅 LMT

降噪完成后，图像进入 SimTool 引擎。普通模式加载 SimTool_Photo_Normal.ini，其 [switch] 段中只有 LMT = 1，其余全为 0。SimTool 在普通模式下只执行一个步骤：LMT（Look Modification Transform，外观修改变换）。

LMT 是色彩管线的"风格塑形层"，目的是在 ISP 输出之后、Gamma 之前，对图像施加轻度的色调调整。普通模式的 LMT 具有以下特征：

首先，**每颗传感器独立配置**（LMTUseSensor = 0/1/2/3/）。固件中存有 Photo/Sensor0/ 至 Photo/Sensor3/ 四个独立目录，每个目录包含 8 张 32³ LMT，每张 384KB，用于补偿各镜头因光学特性不同导致的色彩差异。

其次，更关键的是，**普通模式的 LMT 不做任何色彩风格偏移**。实测全部 8 张 LMT 的色彩响应：对纯红输入（R=1, G=0, B=0），输出为 [1.0, 0.0, 0.0]，没有跨通道注入；对纯绿输入，输出为 [0.0, 1.0, 0.0]，同样没有色偏。灰轴响应接近线性（输入 0.5 → 输出 0.516，基本保持原值）。

这说明普通模式的 LMT 实质上只是一个传感器间的亮度校正层，不承载任何风格设计。普通模式的成像调性完全来自后续的 ISP 硬件 Gamma——而 Gamma 的调校风格是"明亮通透"（详见第 6 章）。

### 3.4 ISP Gamma 曲线

LMT 之后，图像进入 ISP 硬件 Gamma 处理，使用 gamma_photo_hdr_conf.json 定义的曲线。该曲线极为激进地抬升暗部：输入亮度 30（满量程 255 的 11.8%）被映射至输出 409（满量程 1023 的 40%），相当于将该亮度区域提亮约 3.4 倍。这种调校以"暗处也看得清"为目标，在弱光环境下显示效果好，但代价是暗部层次被压扁、照片整体偏向平淡开朗的风格，缺乏阴影的厚重感与立体感。

## 4. 大师模式 JPGmax 管线

大师模式 JPGmax 是 Find X9 Ultra 色彩架构中复杂度最高、设计意图最为明确的拍照路径。它不是在普通模式基础上叠加滤镜，而是从根本上替换了色彩渲染的执行机构：ISP 硬件 Gamma 在此路径中被降格为线性传输通道，色调、色彩、风格的全部控制权由 SimTool 引擎的六个软件模块接管。

### 4.1 传感器分辨率路径：光照自适应的三档读出

Find X9 Ultra 主摄 IMX906 是一颗原生 50MP 的 QuadBayer 传感器，物理像素阵列约为 8192×6144。QuadBayer 结构允许以不同的像素组合方式读出，产生三个不同的分辨率档位。大师模式 JPGmax 会根据光照条件在三档之间自动选择，**最高可输出约 50MP**（实际像素数因逻辑镜头的裁切方式而异，主摄约 50MP，某些焦段因成像圆覆盖面积不同可能为 49MP 或其他接近值）。三档的切换由 ISP 的 Lux Index 自动决定，用户无法手动固定，色彩管线在所有档位下保持不变。

**第一档：Binning 模式（~12.5MP）**

在夜间、深度弱光或高 ISO 场景下，传感器以 2×2 Binning 模式工作，将四个相邻同色物理像素的光信号在模拟域合并，等效像素面积扩大至约 2.0μm，输出约 4096×3072（12.5MP）。这是信噪比最高的读出方式，在极暗场景下是唯一有意义的分辨率选择。大师模式与普通模式在此档使用同级别的降噪模型（binning 系列，3MB 量级）。

**第二档：SQuadBayer 中间分辨率模式（~25MP）**

在光线适中的室内或傍晚场景下，系统使用 SQuadBayer（Smart QuadBayer）或 2× QuadBayer 读出方式，对 QuadBayer 像素进行部分合并而非完全合并，输出约 5600×4200（~23.5MP，通称 25MP）。该档是分辨率与信噪比之间的平衡点。大师模式在此档使用 photo_main_quadbayer_ev0_2x_3f_25M.bin（3.7MB），普通模式使用 photo_main_mix_ev0_25M.bin（2.9MB）。

**第三档：SQuadBayer 全分辨率模式（~50MP）**

在日光等光照充足的条件下，系统切换至 SQuadBayer 全像素读出，输出约 50MP。**这是大师模式 JPGmax 在最佳光照条件下的实际输出分辨率**，对应降噪模型为 main_squadbayer_ev0.bin（3.8MB）。该模型体积与第二档接近（均为 3–4MB 量级），因为 SQuadBayer 全分辨率读出虽然像素数翻倍，但传感器在良好光照下信噪比尚可，不需要哈苏超清专用路径那样庞大的模型（24.6MB）。

哈苏超清使用的 main_quadbayer_ev0_3f_50Mup.bin（24.6MB）是完全不同的另一套模型，针对不做任何合并的纯 QuadBayer 原生读出进行三帧对齐降噪，计算量是大师模式第三档的六倍以上（详见第 5 章）。

### 4.2 AI 降噪模型

大师模式的降噪模型命名带有 `photo_` 前缀（或采用不带前缀的专用命名），区别于普通模式，体现了两套模型针对不同优化目标独立训练的事实：普通模式的降噪模型以速度为优先，大师模式的降噪模型以细节保留为优先，以配合后续高精度 LUT 渲染的要求。三个分辨率档位各有对应的模型组：

**第一档：Binning 12MP 弱光模型**

| 场景 | 模型文件 | 大小 |
|---|---|---|
| 标准曝光，三帧对齐 | photo_main_binning_ev0_3f.bin | 3.1MB |
| 低曝光 | photo_main_binning_evm.bix | 2.6MB |
| 有运动物体 | photo_main_binning_ev0_motion_3f.bin | 3.1MB |
| 低曝光深度弱光 | photo_main_binning_evmm.bin | 2.6MB |

**第二档：SQuadBayer 25MP 中光模型**

| 场景 | 模型文件 | 大小 |
|---|---|---|
| 标准曝光，三帧对齐 | photo_main_quadbayer_ev0_2x_3f_25M.bin | 3.7MB |
| 低曝光 | photo_main_quadbayer_evm_2x_25M.bin | 3.3MB |
| 深低曝光 | photo_main_quadbayer_evmm_2x.bin | 4.4MB |

**第三档：SQuadBayer 全分辨率 ~50MP 日光模型**

| 场景 | 模型文件 | 大小 |
|---|---|---|
| 标准曝光（主摄）| main_squadbayer_ev0.bin | 3.8MB |
| 低曝光（主摄）| main_squadbayer_evm.bin | 4.6MB |
| 深低曝光（主摄）| main_squadbayer_evmm.bin | 4.6MB |
| 标准曝光（3.5× 长焦）| tele1_squadbayer_ev0.bin | 3.8MB |
| 低曝光（3.5× 长焦）| tele1_squadbayer_evm.bin | 4.6MB |

可以注意到，第三档（~50MP）的模型体积（3.8–4.6MB）与第二档（3.3–4.4MB）处于同一量级，均远小于哈苏超清专用的 24.6MB 模型。这说明第三档虽然像素数约为第二档的两倍，但 SQuadBayer 读出方式在良好光照下保证了足够的基础信噪比，不需要像纯 QuadBayer 全像素读出（哈苏超清）那样依赖超重量级的模型来对抗噪点。

### 4.3 LMT 层：哈苏风格的基础塑形

经过降噪处理的图像进入 SimTool，首先经过 LMT（Look Modification Transform）层。大师模式的 LMT 在三个维度上与普通模式的 LMT 有根本性差异。

**精度提升**：大师模式 LMT 精度为 64³，普通模式为 32³。格点体积增加 8 倍（64³ = 262,144 个格点，32³ = 32,768 个），每张 LMT 文件从 384KB 增至 3MB。更高的格点密度意味着 LUT 能精确表达更细腻的局部颜色变化，在高饱和度色彩区域和极端亮度区域的插值误差大幅降低，边缘色彩过渡更为自然连续。

**传感器统一**：大师模式将 LMTUseSensor = 0/ 设为单一值，强制所有镜头（主摄、超广角、长焦、超长焦）共用同一组 LMT，确保无论切换哪颗镜头，输出的哈苏色彩风格都完全一致，不因镜头特性差异而产生调性漂移。

**引入强烈的跨通道色彩耦合**：这是最本质的差异。实测大师模式 LMTLut0（六张 LMT 中的基础调）的色彩响应如下：

| 输入颜色 | 大师模式 LMT 输出 (R, G, B) | 普通模式 LMT 输出 (R, G, B) |
|---|:---:|:---:|
| 纯红 (1, 0, 0) | **(1.00, 0.64, 0.58)** | (1.00, 0.00, 0.00) |
| 纯绿 (0, 1, 0) | **(0.52, 0.97, 0.70)** | (0.00, 1.00, 0.00) |
| 中性灰 (0.50, 0.50, 0.50) | (0.82, 0.82, 0.82) | (0.52, 0.52, 0.52) |

普通模式的 LMT 对纯红输入只输出红通道，绿蓝通道均为 0，完全没有色彩跨通道注入，是纯色保持的。大师模式对纯红输入同时向绿通道注入 0.64、蓝通道注入 0.58——这意味着纯红在 LMT 层就已经被"染色"，不再是纯粹的红色，而是一种混有中间调的橙品色。对纯绿输入，绿通道依然主导（0.97），但红通道注入了 0.52、蓝通道注入了 0.70，纯绿变为带有强烈蓝调的青绿色。

六张 LMT LUT（LMTLut0–5）并非相互独立的风格选项，而是一个以场景亮度（Lux Index）和环境色温（Kelvin）为双坐标轴的插值网格。在拍摄时，SimTool 根据 ISP 报告的当前 Lux 值和色温值，在相邻的两张 LMT 之间进行三维插值，以确保哈苏色彩风格在日光、黄昏、室内白炽灯等不同光照下都保持视觉连贯性。从固件中 TriggerBINPhoto_Sensor0 的二进制结构可以解析出选择阈值：Lux 分界点约为 230 和 250，色温分界点在 3400K、3800K、4400K、4600K 等节点处，构成一个 3×3 的场景矩阵，覆盖从昏暗黄光（3400K 以下）到晴天日光（6000K 以上）的完整范围。

### 4.4 CWCM 层：色温与色调的精密控制

CWCM（Cool-Warm / Cyan-Magenta，冷暖/青洋）是大师模式独有的色彩轴向控制层，普通模式没有这一层。它承担 UI 中"色温"和"色调"两个滑块的底层实现。

该层维护 9 张 LUT（CWCM16Lut0 至 CWCM16Lut8，每张 16³ × FP32 = 48KB），覆盖两个独立的调节轴：

**CoWa 轴（冷暖轴）**：范围 -100 到 +100。这不是简单的色温白平衡偏移，而是通过完整的 3D LUT 实现的非线性色温校正，可以在不同亮度区域施加不同强度的冷暖偏移——暗部和高光区域的偏移量可以独立于中间调，这是线性矩阵白平衡无法实现的精细控制。

**CyMa 轴（青洋轴）**：范围 -100 到 +100。控制色调在绿色（偏青）与品红（偏洋红）方向的偏移，对应传统摄影中白平衡的"绿色偏移"调整，用于补偿荧光灯等特殊光源的色彩污染。

实测冷暖两端的 LUT 灰轴响应清晰揭示了该层的工作机制。以输入中等亮度（0.20）的中性灰为例：最冷端（CWCM16Lut0）输出为 (0.060, 0.231, 0.274)，蓝绿通道大幅提升、红通道强烈压低，画面呈冷蓝调；最暖端（CWCM16Lut8）输出为 (0.285, 0.168, 0.161)，红通道提升、蓝绿压低，画面呈暖橙调；中性位置（CWCM16Lut4）输出与输入完全一致，不引入任何色偏。

用户调节大师模式色温滑块时，SimTool 根据滑块数值在相邻的两张 CWCM LUT 之间做三线性插值，得到连续平滑的色偏效果。在不调整参数、两个滑块均居中的情况下，该层等效于恒等变换，对大师模式的默认成像效果没有影响。CWCM 在 SimTool_Photo_Normal.ini 中被标注为 CMCWLut，同样定义了资源规格（32³，9 张），但 switch 段中被关闭（CWCM = 0），资源虽然加载配置，但实际不运行。

### 4.5 SC 层："影调"滑块的底层机制

SC（Saturation / Contrast，饱和度/对比度）层控制图像的亮度曲线形状，是 UI 中"影调"滑块的底层实现。该层在普通模式中被完全关闭。

SC 层维护 9 张 LUT（SC16Lut0 至 SC16Lut8，每张 16³ × FP32 = 48KB），对应 Cont（对比度/影调）轴从 -100 到 +100 的 9 个均匀分布的锚点。拍摄时，SimTool 根据用户的滑块数值在相邻两张 LUT 之间插值，得到连续的效果。

实测三个代表性锚点的灰轴响应如下，充分说明了影调滑块的实际作用：

| 输入亮度 | SC0（影调 -100，最暗）| SC4（影调 0，中性）| SC8（影调 +100，最亮）|
|:---:|:---:|:---:|:---:|
| 0.20 | 0.112（被压暗 44%）| 0.200（恒等）| 0.327（提亮 64%）|
| 0.40 | 0.271 | 0.400 | 0.562 |
| 0.60 | 0.501 | 0.600 | 0.688 |
| 0.80 | 0.733 | 0.800 | 0.867 |

SC4 的输出与输入完全一致，这是严格恒等变换的标志——默认参数下 SC 层对成像效果没有任何修改。SC0 将中低亮度区域大幅压暗，产生浓郁的阴影密度感；SC8 则将中低亮度大幅提亮，产生"通透高调"的效果，高亮区域则趋近饱和。

SC 层在色彩上是完全中性的：对同一亮度级别，红绿蓝三通道的输出值完全相同，不引入任何色相偏移。这意味着"影调"滑块只改变明暗曲线，不影响色彩倾向。SC 层与 CWCM 层正交独立，两者可以独立调节而互不干扰——调影调不改变色温，调色温不改变影调。

### 4.6 ODT 层：哈苏色彩的核心载体

ODT（Output Device Transform，输出设备变换）是整条色彩管线中意义最重大的层级，位于所有色彩处理的最后阶段，将经过 LMT、CWCM、SC 三层处理后的中间态色彩信号，映射为最终写入 JPEG 的像素值。

固件中共存有四个不同用途的 ODT 文件，清楚揭示了各模式的设计意图：

- **ODT_Pre**（basictone/odt/）：实测为严格线性直通（恒等变换），不改变任何颜色，用作无需色调映射时的 bypass 通道。
- **ODT_Photo**（basictone/odt/）：普通拍照模式使用，具有温和的 S 形对比曲线，不做颜色风格偏移。
- **masterlut/ODT_Photo**（filters_lut/masterlut/）：**大师模式专属**，具有强烈的 S 形对比曲线和显著的跨通道色彩耦合，是哈苏 HNCS 色彩的实现核心。
- **ODT_Dolby**（basictone/odt/）：实测与 ODT_Pre 完全相同（最大像素差 ≈ 0），Dolby Vision 的 HDR 色调映射由容器格式本身处理，ODT 退化为透明通道。

#### 4.6.1 灰轴响应：S 形曲线的量化分析

对中性灰输入（R = G = B），分别测量大师 ODT、普通 ODT 和线性参考的响应，可以清晰看出对比曲线的形状差异：

| 输入亮度 | 大师 ODT 输出 | 普通 ODT 输出 | 线性参考 | 大师相对线性的偏差 |
|:---:|:---:|:---:|:---:|:---:|
| 0.000 | 0.000 | 0.000 | 0.000 | — |
| 0.127 | **0.037** | 0.082 | 0.127 | −71%（暗部深压）|
| 0.254 | **0.171** | 0.200 | 0.254 | −33% |
| 0.381 | **0.366** | 0.334 | 0.381 | −4%（即将反超）|
| 0.508 | **0.605** | 0.467 | 0.508 | +19%（中调大幅抬升）|
| 0.635 | **0.813** | 0.633 | 0.635 | +28% |
| 0.762 | **0.933** | 0.806 | 0.762 | +22% |
| 0.889 | **0.984** | 0.932 | 0.889 | +11%（高光趋近饱和）|



大师 ODT 的灰轴曲线是一条典型的强 S 形：暗部（0.127 区域）被压低至线性值的 29%，而中间调（0.508 区域）被提升至线性值的 119%，高光区域（0.762 以上）快速收敛逼近 1.0。这种曲线在视觉上产生"暗部浓郁、中调通透、高光立体"的厚重感——阴影有密度，亮部有张力，与中画幅胶片的影调特征高度一致。

普通 ODT 的曲线相对温和，只在高光区域有轻微的 S 形弯曲，整体偏向明亮开放的风格。

#### 4.6.2 色彩偏移：跨通道耦合的量化分析

大师 ODT 对颜色的非等比例通道耦合，是其最能体现哈苏色彩哲学的特征。以下数据揭示了大师 ODT 如何对关键色块进行"再诠释"：

| 输入色块 | 大师 ODT 输出 (R, G, B) | 普通 ODT 输出 (R, G, B) | 色彩偏移方向与视觉结果 |
|---|:---:|:---:|---|
| 纯红 (1,0,0) | (1.00, 0.00, **0.26**) | (0.96, 0.00, 0.00) | 蓝通道注入 0.26 → 红色偏向橙品/玫瑰红 |
| 纯绿 (0,1,0) | (0.00, 0.98, **0.47**) | (0.01, 0.99, 0.01) | 蓝通道大量注入 0.47 → 绿色偏向青绿/薄荷 |
| 纯蓝 (0,0,1) | (0.00, **0.17**, 0.75) | (0.00, 0.00, 0.93) | 绿通道注入 + 蓝通道衰减 → 蓝色偏向粉蓝/淡蓝 |
| 肤色暖橙 (~0.63,0.39,0.31) | (1.00, 0.18, 0.19) | (0.58, 0.36, 0.28) | 红通道推至满载 → 肤色极度橙粉化 |
| 天空蓝绿 (~0.31,0.47,0.79) | (0.00, 0.44, 1.00) | (0.28, 0.42, 0.82) | 蓝通道增强 → 天空更纯蓝饱和 |
| 中性灰 (0.50,0.50,0.50) | (0.60, 0.60, 0.60) | (0.47, 0.47, 0.47) | 中调整体提亮，体现 S 曲线中段抬升 |

这些偏移方向精确反映了 HNCS（哈苏自然色彩解决方案）的核心美学：红色向橙品推进，赋予皮肤和暖调景物以厚重饱和感；绿色向青冷倾斜，使植物和草地呈现清冷内敛的绿；蓝色降低饱和度并向粉蓝漂移，使天空显得淡雅而不过度饱和；配合强 S 曲线，整张照片产生出类似中画幅胶片的"厚重、立体、不浮夸"的视觉质感。

#### 4.6.3 ODT 与 LMT 的配合关系

大师 ODT 的输入是经过 LMT、CWCM、SC 三层处理后的中间态色彩信号，而非原始相机 sRGB。这意味着 LMT 已经对颜色施加了初步的哈苏风格偏移（如将纯红推向橙品），ODT 则在此基础上进一步强化并完成最终映射。两者在数学上是配对关系：LMT 做基础风格塑形，ODT 做最终成片渲染，单独使用其中一个而丢弃另一个，都无法复现完整的哈苏色彩效果。

### 4.7 Vig 层：镜头暗角的精准补偿

Vig（Vignette，暗角）层对 Find X9 Ultra 四颗主要镜头的光学暗角进行软件补偿。镜头暗角是所有光学系统固有的缺陷——由于边缘光路较中心更长且存在渐晕效应，图像四角的亮度天然低于中心，程度随焦距和光圈变化。

该层加载 asictone/vig/ 目录下的四张补偿表，分别对应主摄（VigTab_Main）、超广角（VigTab_Wide）、3.5× 长焦（VigTab_Tele）和 6× 超长焦（VigTab_UTele）。每张补偿表是一个 13×17 的二维采样网格，在图像处理时通过双线性插值覆盖全图，对边缘像素的亮度进行精确提升。补偿表按各镜头的实测光学特性单独标定，不可跨镜头通用。

普通模式将 Vig 层完全关闭，其成片可能在大光圈、广角端保留轻微的镜头暗角，而大师模式对此进行精确补偿。由于暗角补偿量随分辨率（图像边缘到中心的比例）不变，同一张补偿表可以在三个分辨率档位下通用，不需要为 12MP/25MP/50MP 分别维护。

### 4.8 Merge 层：多帧合成与 HDR 融合

Merge 层在大师模式中负责将多次曝光或多帧图像合成为最终结果，是普通模式完全缺失的处理步骤。

**多帧 HDR 合成**：在高动态范围场景（如逆光、日落）下，相机连拍数帧不同曝光量的图像，Merge 层将这些帧融合为一张宽容度更高的图像，在同一成片中同时保留暗部细节和高光层次。

**UltraHDR Gain Map 生成**：Find X9 Ultra 支持 UltraHDR 格式（Google 主导的 HDR 图像标准），在 JPEG 文件中嵌入一张亮度增益图（Gain Map）。在支持 HDR 显示的屏幕上查看时，系统读取 Gain Map 并在渲染时提升特定区域的亮度，呈现出超出标准 SDR 显示范围的高光层次。SimTool_Photo_Master.ini 中的 mergeSize = 64 参数控制 Gain Map 的精度格点数。

Merge 层的存在同时解释了大师模式拍照相比普通模式耗时更长的原因——需要完成多帧对齐、融合和 Gain Map 计算，而非单帧直出。

### 4.9 Dither 与 JPEG 编码

所有色彩处理完成后，图像亮度值需要从浮点精度量化为整数（8bit 或 10bit）。量化过程会在平滑渐变区域引入"色带"（banding）——相邻亮度级别之间出现肉眼可见的阶梯状跳变。

Dither（抖动噪声）层通过在量化前向像素亮度注入极细微的受控随机噪声，使量化误差在空间上分散，视觉上表现为均匀的细腻纹理而非连续的色带。噪声数据由 asictone/setting/Ditherout 提供，这是一个 3.1MB 的预计算随机噪声贴图，避免运行时实时生成带来的计算开销。大师模式的 Dither 强度为 Dvig10bit = 5（10bit 路径）和 Dvig8bit = 3（8bit 路径）。

最终，经过 Dither 处理的图像进入 JPEG 编码器，以最高画质编码写入存储。"JPGmax"名称的直接来源正是此处：大师模式的 JPEG 质量系数高于普通模式，压缩比更低，保留更多高频细节，文件体积也更大。

## 5. 大师模式 RAW 与 RAWMax（计算 RAW）管线

在大师模式的 UI 交互中，除了最高画质 JPEG 输出（JPGmax）外，用户还可以选择 **RAW** 和 **RAWMax** 档位。传统认知往往将手机 RAW 误解为“传感器未处理数据的直接转储”，但在 OPPO Find X9 Ultra（PMA110 平台）上，无论是标准 RAW 还是 RAWMax，皆为高度精密、融合了多帧计算与 AI 神经网络降噪的**计算 RAW（Computational RAW）**。

### 5.1 架构定义：TurboRAW 与 HybridRAW 引擎

固件底层的 `odm/etc/camera/` 目录中明确展示了计算 RAW 的双层处理引擎体系：

1. **TurboRAW（RAW 域多帧融合引擎）**
   - **核心逻辑**：在硬件或内核驱动层运行的多帧对齐与合成管线。当快门触发时，传感器进行极速连拍，获取包含短曝光（`RAWShort`）、中曝光（`RAWMiddle`）与长曝光的多个 RAW 原始数据流。在未去马赛克（Demosaic）之前，系统直接在拜耳域（Bayer Domain）执行光流对齐与多帧合成。
   - **配置特征**：在 `config/lighthousemain` 等配置文件中，表明 TurboRAW 支持零延迟连拍（Frameless 架构）：
     ```ini
     IS_SUPPORT_TURBORAW_FRAMELESS = 1,1,1,1,1,1
     ```
     此外，该引擎集成了运动防抖与检测机制（`TURBOERAW_SUPPORT_MOTION_SCENE`）以及夜景暗电流补漏光校正（`NIGHT_AI_PURERAW_ENABLE_LSC = 1`），确保输出一帧动态范围极宽、信噪比极其优异的“干净底片”。

2. **HybridRAW（计算 RAW / 16-bit DNG 封装引擎）**
   - **核心逻辑**：在 TurboRAW 完成多帧合并后，数据通过 `OplusSATPhoto.xml` 中的 `RAWOutPort_RAW` 节点送入 **HybridRAW** 引擎。系统调用 `hybridraw_models/` 下的专用 AI 神经网络（如超分辨率模型与 RAW 域双向去噪模型）进行深度净化。
   - **最终输出**：将深度处理后的 16-bit 线性像素流与哈苏大师调色标定参数封装在一起，生成兼容 Adobe DNG 1.6 标准的通用计算 RAW 文件。

### 5.2 分辨率规格与动态场景 ISO 触发门限

不同于普通 RAW 固定在 12.5MP 输出，大师模式 RAWMax 提供了传感器全像素输出能力的完全释放：

* **标准 RAW / 融合基准分辨率**：
  在 `lighthousetele`（长焦）、`lighthouseultratele`（超长焦）和 `lighthousemain`（主摄）的配置中，标准融合计算分辨率均统一标定为：
  ```ini
  TURBO_HDR_RAW_WIDTH = 4096
  TURBO_HDR_RAW_HEIGHT = 3072
  ```
  即标准的 12.5MP（4096 × 3072）四合一 Binning 分辨率。
* **RAWMax 全像素输出**：
  当光线条件满足且用户开启 RAWMax 时，系统在 SQuadBayer 模式下直接绕过后端缩减节点，调用全像素解包路径，输出高达 **50MP**（主摄约 8192 × 6144，实际有效分辨率依镜头视角与几何校正略有裁切）的超高清 16-bit 线性 DNG 文件。

系统通过环境照度（Lux Index）与感光度（ISO）自动调控多帧计算策略：

| 触发参数名称 | 主摄（Main） | 长焦（Tele 3.5×） | 物理意义与控制逻辑 |
|---|:---:|:---:|---|
| `RAWSR_LUX_INDEX_TRIGGER` | 290 | — | 触发 RAW 域神经网络超分辨率（RAWSR）的环境亮度门限 |
| `NORMAL_RAWHDR_OUTDOOR_ISO` | 160 | 210 | 户外高动态光照下，触发 RAW 域多帧 HDR 融合的 ISO 阈值 |
| `NORMAL_RAWHDR_INDOOR_ISO` | 400 | 330 | 室内中低光照下，触发 RAW 域多帧 HDR 融合的 ISO 阈值 |
| `HQRAW_ISO_DELAY_SWITCH` | 4950, 5050 | 2950, 3050 | 高感光度极限下，启动高质量降噪 RAW（HQRAW）的迟滞切换门限 |
| `HQRAW_LUX_DELAY_SWITCH` | 375, 385 | 375, 385 | HQRAW 启动的光照照度延时切换门限 |
| `HQRAW_DRCGAIN_TRRIGGER` | 3.2 | 1.5 | 动态范围控制（DRC）硬件增益触发门限 |

当环境照度低于 `RAWSR_LUX_INDEX_TRIGGER`（290 Lux）时，AI RAW 超分模型会自动启动以补充细节；而在高 ISO 夜景场景（如主摄 ISO > 4950），系统则启动 HQRAW 模式并大幅提高硬件 DRCGain 增益，防止暗部涌现噪点。

### 5.3 DNG 颜色校准系统与哈苏色彩映射

大师模式 RAWMax 之所以能在 Lightroom、Photoshop 或 DaVinci Resolve 中开箱即获得极高水准的“哈苏色彩”，是因为 OPPO 在固件的 `odm/etc/camera/dng/` 目录中，为每一颗镜头精心烧录了 Adobe DNG 规范的超高精度色彩标定资产。

#### 1. 双光源基础颜色矩阵（ColorMatrix1 / ColorMatrix2）
DNG 规范通过两个不同色温的标准光源矩阵，将原始传感器色彩空间映至绝对 XYZ 色彩空间。第三方软件运行时根据实际拍摄色温在两者间执行线性插值：
* **低色温光源（A 光源 / Tungsten 钨丝灯，标定色温 ~2856K）**
  主摄文件 `ColorMatrix1_lighthousemain.txt` 精切实测浮点矩阵：
  $$\mathbf{M}_1 = egin{bmatrix} 0.79581 & -0.29316 & -0.05997 \ -0.41033 & 1.22984 & 0.15173 \ -0.16292 & 0.31327 & 0.49580 \end{bmatrix}$$
  *(超广角 `wide` 为 `[0.7180, -0.1983, -0.0760...]`；长焦 `tele` 为 `[0.8144, -0.2936, -0.0681...]`)*
* **高色温标准光源（D65 日光，标定色温 ~6504K）**
  主摄文件 `ColorMatrix2_lighthousemain.txt` 精切实测浮点矩阵：
  $$\mathbf{M}_2 = egin{bmatrix} 1.07171 & -0.48973 & 0.03617 \ -0.42422 & 1.38891 & 0.24719 \ -0.10034 & 0.26209 & 0.68748 \end{bmatrix}$$
  *(长焦 `tele` 为 `[1.0267, -0.4194, -0.0583...]`)*

#### 2. 3D 色彩微调网格（HueSatMap）
为了实现哈苏自然色彩解决方案（HNCS）中“红偏橙品、绿偏青冷、深邃蓝天”的非线性色彩美学，单纯的 3×3 线性矩阵远远不够。固件在 DNG 中嵌入了海量的 3D 色调-饱和度微调表：
* 低色温表：`D28HUESATMap_lighthouse*.txt`（~400KB）
* 日光表：`D65HUESATMap_lighthouse*.txt`（~330KB）

每个文件精确包含 **43,200 个浮点数**，在 DNG 结构中对应：
$$	ext{HueDivisions} (120) 	imes 	ext{SatDivisions} (20) 	imes 	ext{ValDivisions} (2) 	imes 3 (\Delta H, \Delta S, \Delta V) = 43,200$$
这表示哈苏调色团队把色相环极细致地切割为 120 份、饱和度切割为 20 份、明度分为 2 层，为高达 2400 个基准颜色区域独立标定了色相旋转（$\Delta H$）、饱和度缩放（$\Delta S$）和明度缩放（$\Delta V$）。这就是为何在使用第三方软件打开 RAWMax DNG 时，无需应用任何额外预设，便已天然具备极具厚重感与立体感的哈苏高级色彩特征。

### 5.4 DNG 默认光影曲线（ToneCurve）

DNG 像素流在数学上是纯线性的（Gamma = 1.0）。若不加反差映射，直接在屏幕查看会显得极其灰暗且反差极低。为此，固件定义了 `ToneCurve_lighthouse*.txt`，包含了 256 对灰阶插值点。



这一条经过特殊优化、富有胶片呼吸感的 S 曲线作为 `DefaultUserCrop` 默认色调曲线嵌入 DNG 文件头。它在阴影区（0.05 ~ 0.25）给出了适度的线性上抬以显露细节，在中调区域保持高反差立体感，而在高光区（> 0.80）则执行极其丝滑的柔性渐近收敛。这使得 RAWMax 即便直接出图，也具备可媲美大师模式 JPGmax 的精致明暗结构，同时保留了 16-bit 线性底片那宽广无垠的后期二次调色空间。

## 6. 哈苏超清（50MP）管线

### 5.1 定义与定位

哈苏超清（50MP）是 Find X9 Ultra 的极限分辨率拍摄路径，在 UI 上以独立入口提供，与大师模式 JPGmax 的分辨率档位有所重叠（均可输出 ~50MP），但两者的 AI 处理链路截然不同，针对不同的使用场景进行了优化。

### 5.2 传感器读出：纯 QuadBayer 全像素原生读出

哈苏超清模式令 IMX906 以纯 QuadBayer 全像素原生方式读出，不做任何像素合并，直接产出约 8192×6144 的完整 RAW 数据。与大师模式第三档的 SQuadBayer 读出不同，这种模式下每个物理像素完全独立读出，像素间没有任何信号合并。

代价是每个像素的等效面积只有约 1.0μm（Binning 模式的 1/4），进光量大幅下降，信噪比在同等光照条件下显著劣化。即使在日光下，每个独立小像素的信噪比也远不如 SQuadBayer 合并后的效果，这是哈苏超清需要超大 AI 降噪模型的根本原因。

### 5.3 AI 降噪：重量级三帧对齐模型

哈苏超清的核心差异在于其 AI 降噪模型的规模：

| 模型文件 | 大小 | 作用 |
|---|---|---|
| main_quadbayer_ev0_3f_50Mup.bin | 24.6MB | 主摄 50MP 三帧对齐降噪，标准曝光 |
| main_quadbayer_ev0_3f_person_50Mup.bin | 24.6MB | 主摄 50MP 三帧降噪，人像场景专用 |
| tele1_quadbayer_ev0_3f_50Mup.bin | 24.7MB | 3.5× 长焦 50MP 三帧降噪 |
| jdd_enhancernet_50m.bin | 704KB | 50MP 专用联合去马赛克增强网络 |
| ai_flow_back_50M.bix | 2.3MB | 50MP 分辨率光流估计（帧间对齐）|
| ai_motion_mask_50M.bix | 4.6MB | 50MP 运动区域检测（防止运动帧引入伪影）|

24.6MB 的主模型是大师模式第三档 main_squadbayer_ev0.bin（3.8MB）的 6.5 倍，体积的差距直接反映了计算量和网络深度的差距。该模型利用连续三帧的原始 QuadBayer 数据做时域对齐降噪（3f = 3 frames），通过 ai_flow_back_50M.bix 估计帧间的亚像素位移，并用 ai_motion_mask_50M.bix 识别运动区域避免残影，最终输出单张高信噪比的 50MP 图像。

jdd_enhancernet_50m.bin（JDD = Joint Demosaicing & Denoising，联合去马赛克与降噪）将"把 QuadBayer RAW 还原为彩色图像"和"去除剩余噪点"两个步骤合并进一个 AI 网络同时完成，相比先 demosaic 再降噪的分步处理，避免了中间步骤引入的误差积累。

### 5.4 色彩管线：共享当前拍摄档位

哈苏超清的 AI 降噪链路完成后，图像进入 SimTool 色彩引擎。此处的关键特性是：**哈苏超清没有独立的色彩管线**，它共享当前所选拍摄模式的 SimTool 配置。

- 若用户在大师模式下触发哈苏超清：使用 SimTool_Photo_Master.ini，完整的六模块哈苏色彩管线全部运行。
- 若用户在普通模式下触发哈苏超清：使用 SimTool_Photo_Normal.ini，只有 LMT 运行，无 ODT，无哈苏色彩。

这一设计意味着"哈苏超清"标签描述的是分辨率路径，而非色彩路径。哈苏色彩来自大师模式的 SimTool 配置，与是否选择 50MP 全像素读出无关。

### 5.5 与大师模式 JPGmax 的实质差异对比

| 对比项 | 大师模式 JPGmax（光照充足时）| 哈苏超清（50MP）|
|---|---|---|
| 传感器读出方式 | SQuadBayer（部分合并）| 纯 QuadBayer 全像素 |
| 单像素等效面积 | ~1.4μm（合并后）| ~1.0μm（全像素）|
| AI 降噪模型大小 | 3.8MB（main_squadbayer_ev0）| 24.6MB（50Mup）+ 7MB 辅助模型 |
| 三帧时域对齐 | 无（单帧或快速帧）| 有（3 帧精确对齐）|
| 拍摄耗时 | 快（模型轻量）| 慢（模型超大，对齐计算重）|
| 色彩管线 | 完整大师 SimTool | 取决于所选拍摄档位 |
| 输出分辨率 | ~50MP（良好光照下）| ~50MP |
| PostSR 后处理 | 标准混合（50% 权重）| 极低混合（1% 权重，大模型已充分处理）|
| 设计目标 | 速度与画质的高档平衡 | 极限静态画质，不考虑速度 |

## 7. Gamma 曲线：两种截然不同的设计哲学

### 7.1 Gamma 的作用与位置

Gamma 曲线是 ISP 硬件处理链的最后一道关卡，在 SimTool 软件管线之后（或在 ODT 缺席时替代 ODT）运行。它的作用是将图像的线性亮度值转换为符合人眼感知特性的非线性编码：人眼对暗部亮度变化更为敏感，Gamma 曲线通过"压缩高光、保留暗部"的非线性弯曲来优化动态范围的主观利用率。

普通模式和大师模式各自使用一套专属的 Gamma 曲线配置，两者的设计哲学截然对立，分别服务于各自的成像目标。

### 7.2 普通模式 Gamma：激进提亮的"通透"哲学

普通模式使用 gamma_photo_hdr_conf.json，曲线特征是对暗部区域的极度激进提亮：

| 输入亮度（0–255 编码）| 输出亮度（0–1023 编码）| 提亮比例 |
|:---:|:---:|:---:|
| 0（纯黑）| 0 | — |
| 30（深暗，11.8%）| 409（40.0%）| +239% |
| 60（暗部，23.5%）| 643（62.8%）| +167% |
| 120（中暗，47.1%）| 783（76.6%）| +63% |
| 180（中亮，70.6%）| 846（82.7%）| +17% |
| 240（高光，94.1%）| 889（86.9%）| −8%（轻微压缩）|

输入 30（深暗区域）被映射至输出 409——这意味着人眼看来相当暗的区域，在成片中被拉到了中等亮度。整条曲线的效果是将直方图整体向右大幅推移，在任何光线条件下，成片都显得"明亮通透"，暗处细节总能看清。

这种设计的代价是暗部对比度被大幅压扁——阴影内部的层次感丧失，照片整体缺乏立体感和厚重感。这是一种以牺牲"光影深度"换取"全域可见度"的工程取舍，适合大多数日常使用场景，但难以产生具有艺术感的影调层次。

### 7.3 大师模式 Gamma：接近线性的"交权"哲学

大师模式使用 gamma_masterMode_quick_hdr_conf.json，曲线接近线性：

| 输入亮度（0–255 编码）| 输出亮度（0–1023 编码）| 提亮比例 |
|:---:|:---:|:---:|
| 0（纯黑）| 0 | — |
| 30（深暗）| 120 | +33%（几乎不提亮）|
| 60（暗部）| 240 | 线性 |
| 120（中暗）| 480 | 线性 |
| 180（中亮）| 720 | 线性 |
| 240（高光）| 960 | 线性 |



大师模式的 Gamma 曲线几乎就是一条直线：输入 30 的深暗区域只被映射至输出 120，仅有 33% 的提亮，而普通模式同一输入提亮了 239%。整条曲线保持接近 4：1 的线性比例关系，几乎不做任何非线性弯曲。

这种设计的意图不是"放弃处理"，而是"将暗部的控制权完整交给后续的 ODT 层"。大师模式的 ODT（哈苏 3D LUT）承担了全部的非线性影调塑形职责，其 S 形曲线在暗部的强力压制（将 0.127 的输入压至 0.037 的输出）正好接管了 Gamma 释放出来的这部分控制权。两者共同构成完整的影调设计：Gamma 保持线性以不干扰，ODT 的 S 曲线精确地从零开始塑造整个影调层次。

如果在大师模式下使用普通模式的激进 Gamma，ODT 的 S 曲线就会叠加在已经被大幅提亮的暗部信号之上，导致暗部在高光方向爆掉，无法产生哈苏特征的阴影密度感。两套 Gamma 与各自的 ODT 在数学上是配对调校的，不可混用。

## 9. HNCS 与大师模式的技术关系

### 9.1 HNCS 是什么

HNCS（Hasselblad Natural Colour Solution，哈苏自然色彩解决方案）是瑞典中画幅相机品牌哈苏的专有色彩调校哲学，最初来源于哈苏中画幅数码相机（如 H6D、X2D 系列）多年积累的色彩调校经验。其核心美学主张可以归纳为以下几个维度：

- **红色偏向橙品**：对发色、皮肤、暖调景物赋予厚重的橙粉饱和感，不使用纯正红色。
- **绿色偏向青冷**：植物、草地呈清冷内敛的薄荷绿，避免过于鲜艳的荧光绿。
- **蓝色去饱和化**：天空呈淡雅的粉蓝，不过度饱和，保留空气感。
- **强 S 形对比曲线**：暗部有浓郁密度，中间调通透立体，高光快速收敛，整体质感接近中画幅胶片。
- **不浮夸**：整体色彩表现厚重而非艳丽，更接近胶片的有机质感而非数字直出的鲜艳平滑。

### 9.2 HNCS 在大师模式中的载体

大师模式中的"哈苏色彩"不是运行在设备上的独立哈苏软件，也不是一层简单叠加的后期滤镜。其真正的技术载体是固件中的一批 LUT 文件，以 ilters_lut/masterlut/ODT_Photo（3MB，64³）为核心，由哈苏工程师与 OPPO 联合调校后直接烧录于固件。

从前述章节的实测数据中，可以逐项验证 HNCS 特征在大师 ODT 中的数学体现：

| HNCS 色彩主张 | 大师 ODT 实测数据 | 是否吻合 |
|---|---|:---:|
| 红色偏向橙品 | 纯红输入 → 蓝通道注入 0.26，红色偏向玫瑰橙品 | ✓ |
| 绿色偏向青冷 | 纯绿输入 → 蓝通道注入 0.47，绿色偏向薄荷青 | ✓ |
| 蓝色去饱和化 | 纯蓝输入 → 饱和度降低，绿通道注入，变粉蓝 | ✓ |
| 肤色橙粉夸张 | 肤色输入 → 红通道推至 1.00 满载 | ✓ |
| 强 S 形对比 | 暗部 0.127 → 输出 0.037（线性值的 29%），中调 0.508 → 输出 0.605（+19%）| ✓ |

五项特征全部从数据层面得到验证。这证实了大师模式的哈苏色彩并非营销标签，而是由具体的数字映射关系实现的——这些映射关系被编码在 3MB 的二进制 LUT 文件中，在每次大师模式拍照时由 SimTool 引擎加载并执行。

### 9.3 大师模式与普通模式的根本性差别

从架构层面看，两种模式的差距不是"哈苏色彩强度多少"的问题，而是"整个色彩渲染路径是否存在"的问题：

普通模式的成像色调，从根本上由 ISP 芯片的 Gamma 硬件电路决定。Gamma 是一条对所有颜色一视同仁的单维亮度曲线，它处理红绿蓝三个通道完全对称，不引入任何色彩风格倾向。普通模式是一个追求"准确还原"的中性系统。

大师模式的成像色调，由哈苏调校的专属 3D LUT 在软件层面完全接管。3D LUT 可以对每个输入颜色施加独立的、非等比例的通道变换，将红绿蓝三个轴上的颜色分别推向不同的目标。大师模式是一个具有明确艺术风格取向的有主张系统。

用一句话总结：**普通模式的色彩由 ISP 芯片的物理电路决定；大师模式的色彩由哈苏与 OPPO 联合签署的数学映射表决定，这张表烧录在固件中，是 HNCS 色彩哲学在 Find X9 Ultra 上的物质形态。**

### 4.10 PostSR：后处理超分增强

大师模式 JPGmax 管线中存在一个鲜少被提及但确实运行的最终步骤：PostSR（Post Super-Resolution，后处理超分辨率增强）。它在 Dither 与 JPEG 编码之前运行，对图像施加轻量的卷积神经网络推理，进一步恢复和强化图像细节。

**模型文件**

固件中存有两个 PostSR 模型：

| 文件 | 大小 | 适用场景 |
|---|---|---|
| postSr.bin | 2.6MB | 标准路径（12MP / 25MP / ~50MP SQuadBayer）|
| postsr_2x.bin | 788KB | 2× 上采样路径（用于需要将图像放大至更高分辨率的场景）|
| gan_enhancer_25m_tinny.bin | 544KB | 25MP 专用轻量 GAN 增强器 |

**混合权重机制**

PostSR 不直接以网络输出替换原图，而是将网络输出与原始图像按权重融合，conf_tuning_params.json 中的 PostSRParamsSet 记录了三套权重配置：

| 参数集 | BlendWeight（通用）| FaceBlendWeight（人脸区域）| NormalBlendWeight（平坦区域）|
|---|:---:|:---:|:---:|
| Is50MParams（50MP 哈苏超清路径）| 0.50 | 0.30 | 0.50 |
| Is200MParams（极限变焦路径）| 0.50 | 0.30 | 0.80 |
| IsLargeModelParams（大模型路径，如 24.6MB 模型后）| 0.01 | 0.01 | 0.20 |

BlendWeight 表示 PostSR 网络输出在最终图像中所占的比例，原图占 1 - BlendWeight。

在大师模式 JPGmax 的标准路径（使用 3–4MB 量级的 SQuadBayer 或 binning 降噪模型）下，PostSR 以约 0.5 的权重进行混合，意味着最终图像是"原始降噪结果"与"PostSR 增强结果"各占一半的融合体。在人脸区域（FaceBlendWeight = 0.30），混合权重降低至 30%，以避免超分处理在皮肤纹理上产生过于人工化的磨皮效果。在哈苏超清的 24.6MB 大模型路径后，PostSR 权重几乎为零（0.01），因为重量级 AI 降噪模型自身已经做了充分的细节恢复，PostSR 的追加效益极低，几乎退化为透明通道。

**PostSR 的物理意义**

PostSR 的设计目标是补偿降噪过程中不可避免的细节损失。AI 降噪模型在抑制噪点的同时，也不可避免地会软化图像中的高频细节（边缘、纹理、毛发）——这是降噪与锐度之间的基本矛盾。PostSR 专门针对这种"降噪后的软化"进行补偿，在不重新引入噪点的前提下，以卷积方式恢复图像中的结构性高频信息。

从最终成片体验来看，大师模式 JPGmax 的高锐度感，一部分来自高质量的降噪模型（保留了更多原始细节），另一部分来自 PostSR 的后处理增强。这两个模块共同构成了"高画质 JPEG"这一定位的技术支撑。

## 10. 完整管线流程图

以下两张流程图汇总了大师模式 JPGmax 与普通模式的完整处理链，便于整体对照。

### 10.1 大师模式 JPGmax 完整链路

`
传感器（IMX906 QuadBayer 原生 50MP）
    │
    ├─── 弱光（EV-）→ 2×2 Binning → ~12.5MP RAW
    ├─── 中光（EV0 中等）→ SQuadBayer 2× → ~25MP RAW
    └─── 强光（EV0 日光）→ SQuadBayer 全读 → ~50MP RAW
    │
    ▼
AI 降噪（binning 系列 ~3MB / quadbayer_2x_25M 系列 ~3.7MB / squadbayer 系列 ~3.8MB）
    │  三帧对齐（ev0_3f）、光流估计（ai_flow_back.bix）、运动检测（ai_motion_mask.bix）
    ▼
ISP 硬件预处理
    │  AWB 白平衡校正、Demosaic 去马赛克、AE 曝光确认
    │  Gamma：gamma_masterMode_quick_hdr_conf.json（接近线性）
    ▼
SimTool 引擎（SimTool_Photo_Master.ini）
    │
    ├─[1] LMT 层（64³ × 6张，统一传感器）
    │      场景/色温自适应选择 LMTLut0–5，三线性插值
    │      → 引入哈苏风格跨通道色彩耦合（红偏橙品，绿偏青冷）
    │
    ├─[2] CWCM 层（16³ × 9张）
    │      根据 UI 色温/色调滑块在 CWCM16Lut0–8 间插值
    │      → 非线性冷暖/青洋色偏控制
    │
    ├─[3] SC 层（16³ × 9张）
    │      根据 UI 影调滑块在 SC16Lut0–8 间插值
    │      → 纯亮度曲线变换（不影响色相）
    │
    ├─[4] ODT 层（64³ × 1张，masterlut/ODT_Photo，3MB）
    │      最终色调映射：强 S 曲线 + HNCS 哈苏色彩偏移
    │      → 暗部深压、中调大幅抬升、高光收敛、色彩特征化
    │
    ├─[5] Vig 层（13×17 网格，按当前镜头选 VigTab_*）
    │      补偿镜头光学暗角
    │
    └─[6] Merge 层
           多帧 HDR 合成 + UltraHDR Gain Map 生成
    │
    ▼
PostSR（postSr.bin，2.6MB）
    │  与原图按权重融合（通用 50%，人脸 30%）
    │  → 恢复降噪损失的高频细节
    ▼
Dither（Ditherout 噪声贴图，500×1000）
    │  防止量化色带
    ▼
JPEG 编码（最高画质 JPGmax）
    │
    └─→ 成片（~50MP / ~25MP / ~12.5MP，取决于光照条件）
`

### 10.2 普通模式完整链路（对照）

`
传感器（IMX906 QuadBayer）
    │
    ├─── 弱光 → 2×2 Binning → ~12.5MP RAW
    └─── 强光 → SQuadBayer / Mix → ~25MP RAW
    │
    ▼
AI 降噪（binning 系列 ~3MB / mix 系列 ~2.9MB）
    │
    ▼
ISP 硬件预处理（AWB / Demosaic / AE）
    │  Gamma：gamma_photo_hdr_conf.json（激进提亮，暗部 +239%）
    ▼
SimTool 引擎（SimTool_Photo_Normal.ini）
    │
    └─[1] LMT 层（32³ × 8张，per-sensor 独立）
           仅做轻度亮度校正，无色彩风格偏移
           （SC / CWCM / ODT / Vig / Merge 全部关闭）
    │
    ▼
PostSR（postSr.bin，轻度混合）
    ▼
Dither
    ▼
JPEG 编码（标准画质）
    │
    └─→ 成片（~25MP / ~12.5MP）
`
